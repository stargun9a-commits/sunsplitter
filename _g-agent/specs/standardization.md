# 📏 SOVEREIGN STANDARDIZATION PROTOCOL (V1.1)

## 1. OUTPUT & FORMATTING STANDARDS
To maintain maximum information density and minimize token sprawl:
- **Single-Space Mandate**: No more than one empty line (1 white space) between headers, paragraphs, or sections.
- **No Separators**: Avoid using `---` horizontal rules unless strictly required for multi-document parsing.
- **Direct Inception**: Present tables and code blocks immediately following their descriptive header.

## 2. FILENAMING CONVENTIONS (SKY-CRANE)
All active technical implementation and build logic files reside in `[project-root]/enginer/`:
- **Requirements**: `wire-specs-[MMDD]-[HHMM].md`
- **Directives**: `prompt-[MMDD]-[HHMM].md`
- **Build Reports**: `wire-in-[MMDD]-[HHMM].md`

## 3. DISCOVERY CONVENTIONS (GRID-SCAN)
All active forensic research and protocol-scanning files reside in `[project-root]/discovery/`:
- **Targeting**: `target-scan-[MMDD]-[HHMM].md`
- **Audits**: `forensic-audit-[MMDD]-[HHMM].md`

## 4. ARCHIVAL & GROUNDING
- **Permanent Storage**: Duplicate all validated reports to `/media/conrad/GRAVITY_DATA/REPORTS/[PROJECT-NAME]/`.
- **Hardware Awareness**: Always reference `sys-specs.md` for environmental constraints.
- **Forensic Reference**: Utilize `PANDORA_FORENSIC_REFERENCE.md` for all breach audits.
